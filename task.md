## 2.2 Вычислительная ниткография

Техника изготовления картин из ниток — ниткография (англ. string art) — заключается в создании произведения искусства с использованием однотонных или разноцветных ниток. Известно несколько техник ниткографии, значительно различающихся по художественному эффекту. Нитки могут использоваться в качестве кисти или непосредственно формировать изображение. В последнем случае нитки могут наклеиваться, формируя произвольные узоры, либо вышиваться или натягиваться, образуя исключительно прямолинейные отрезки. В 2016 году художник-экспериментатор П. Вреллис своими работами вызвал волну интереса к художественной технике XXI века — вычислительной ниткографии.

Рис. 2.1: Пример входного изображения и построенного по нему Ани Абакумовой портрета из ниток.

В варианте Вреллиса ниткография формируется нитями, закрепленными на раме картины, и пересекающими изображение «от края до края» под произвольными углами. Картина строится по какому-либо эталонному изображению, выполненному в иной технике, то есть вычислительная ниткография в первую очередь — техника репродукции. На Рис. 2.1 слева показан фрагмент картины Леонардо да Винчи «Мона Лиза», а справа — построенная по нему Аней Абакумовой картина из ниток.

Схему расположения нитей в вычислительной ниткографии рассчитывают на компьютере с помощью алгоритма, который можно построить несколькими способами. В данной задаче требуется построить такой алгоритм на основе одного известного метода вычислительной рентгеновской томографии (ВРТ, computed tomography).

В основе метода ВРТ лежит интегральное преобразование Радона, которое ставит в соответствие непрерывной финитной функции ( f(x, y) ), определенной на ( \mathbb{R}^2 ), множество ее интегралов вдоль всевозможных прямых. При нормальной параметризации прямых результат преобразования Радона записывается как функция в координатах ( (\rho, \varphi) ):

[
R[f(x, y)](\rho, \varphi) =
\int_{-\infty}^{\infty}
f(\rho \cos \varphi - l \sin \varphi,,
\rho \sin \varphi + l \cos \varphi),dl,
\tag{2.2}
]

где ( \varphi ) — угол наклона прямой, вдоль которой вычисляется интеграл, а ( \rho ) — расстояние от нее до начала координат. Радон-образ в нормальной параметризации часто называют синограммой, поскольку образом точки в этом случае является отрезок синусоиды. На Рис. 2.2б показана синограмма изображения, приведенного на Рис. 2.2а. Проекцией функции ( f(x, y) ) под углом ( \varphi ) называется сечение (подмножество) ( P_\varphi(\rho) ) Радон-образа, содержащее интегралы вдоль всех прямых под этим углом:

[
P_\varphi(\rho)=R(\rho,\varphi),
]

то есть Радон-образ — это набор проекций под всевозможными углами.

Как же связаны рентгеновская томография и преобразование Радона? Рассмотрим рентгенограмму какого-либо объекта — фотографию «на просвет», сделанную в параллельном пучке рентгеновских лучей. Зафиксируем любое сечение нашего объекта, перпендикулярное плоскости рентгенограммы. Введем в этой плоскости функцию ( f(x, y) ) пространственного распределения линейного коэффициента поглощения рентгеновского излучения, иначе говоря — функцию оптической плотности объекта. Проекцией рассматриваемой плоскости на рентгенограмму будет прямая. Можно показать, что логарифм облученности, регистрируемый рентгеновским детектором при формировании рентгенограммы, на этой прямой будет равен проекции ( P_\varphi(\rho) ) функции ( f(x, y) ), причем угол ( \varphi ) зависит от взаимного расположения используемых координатных систем. При вращении объекта вокруг оси, перпендикулярной рассматриваемому сечению, на детекторе будет последовательно зарегистрирован радоновский образ ( R[f(x,y)](\rho,\varphi) ). Задачей ВРТ является восстановление функции ( f(x,y) ) оптической плотности исследуемого объекта по дискретной выборке из ее синограммы ( R[f(x,y)](\rho,\varphi) ), регистрируемой детектором при просвечивании рентгеновскими лучами.

Классический алгоритм решения задачи ВРТ — «свертка и обратное проецирование» (Filtered Back Projection, FBP). Он состоит из двух шагов:

1. Свертка каждой проекции ( P_\varphi(\rho) ) с рамп-фильтром [5].
2. Обратное проецирование: каждая полученная на предыдущем шаге одномерная функция равномерно «размазывается» по соответствующему направлению на все реконструируемое изображение; полученные изображения суммируются.

На Рис. 2.2в показан результат реконструкции синограммы, изображенной на Рис. 2.2б, этим алгоритмом. Алгоритм FBP дает довольно точный результат реконструкции даже при небольшом числе проекций. На Рис. 2.2г показан результат реконструкции той же синограммы, предварительно прореженной по числу углов в 4 раза.

Подробное описание алгоритма FBP приведено, в частности, в работе [6]. Здесь же мы для справки приведем аналитическое выражение для рамп-фильтра ( F_r(\rho) ) в пространстве проекций с дискретным шагом по ( \rho ):

[
F_r(\rho)=
\begin{cases}
1/4, & \rho=0, \
0, & \rho \text{ четно}, \
-1/(\pi \rho)^2, & \rho \text{ нечетно},
\end{cases}
\tag{2.3}
]

а также приведем поясняющие иллюстрации для этапа обратного проецирования.

Рис. 2.3: Результат обратного проецирования: а) одной проекции для угла (60^\circ); б) трех проекций — для углов (60^\circ), (105^\circ) и (150^\circ).

Из этих иллюстраций ясно, что операция обратного проецирования может быть физически промоделирована протягиванием параллельного пучка нитей заданных оттенков серого. Разумеется, светлоты нитей, идущих под разными углами, при наложении не будут суммироваться — нити переднего плана полностью заслонят все остальные. Но если толщина нити значительно меньше стороны пикселя, то средняя яркость пикселя, через который проходит несколько нитей, будет незначительно отличаться от линейной модели, поскольку площадь пересечения нитей мала по сравнению с общей занимаемой ими площадью. Поэтому будем считать модель суммации адекватной.

Будем строить ниткографическое изображение на черном фоне. Пусть у нас есть нить одного цвета — белого. Даже в таких условиях алгоритм вычислительной ниткографии позволит нам построить узнаваемую репродукцию. Чтобы достичь этого, воспользуемся случайным распылением (random dithering). В случае, если требуется «обратно спроецировать» нить со светлотой ( a ) ((0 < a < 1)), с вероятностью ( a ) протянем белую нить, а с вероятностью ( 1-a ) — оставим изображение неизменным.

Следует заметить, что, хотя Радон-образ от неотрицательного изображения также неотрицателен, проекции после свертки с рамп-фильтром могут содержать довольно много отрицательных значений. Эту проблему можно устранить поднятием общей яркости исходного изображения, но мы для простоты будем ограничивать значения проекций нулем снизу.

Окончательно сформулируем томографический алгоритм вычислительной ниткографии. Алгоритм принимает на вход изохромное (одноканальное, в оттенках серого) изображение, а также желаемое число углов, под которыми натянуты нити, и максимальное число нитей, протянутых в каждом направлении. Выход алгоритма — схема расположения прямых в картине из ниток. Шаги алгоритма и их иллюстрации приведены ниже:

1. Вычисление преобразования Радона для заданного числа углов (результат показан на Рис. 2.4а);
2. Свертка проекций с рамп-фильтром (Рис. 2.4б);
3. Прореживание проекций по пространственной переменной до заданного числа нитей (Рис. 2.4в);
4. Нормировка: занулим точки синограммы, значения которых меньше 0; нормируем оставшийся диапазон к отрезку ([0,1]);
5. Бинарное распыление синограммы: в каждом пикселе выходной синограммы поставим 1 с вероятностью, указанной в этом пикселе, иначе поставим 0 (Рис. 2.4г);

Для проверки работы программы, реализующей основной алгоритм, требуется программа визуализации, моделирующая натягивание нитей. Ее алгоритм достаточно очевиден: требуется построить обратную проекцию данной бинарной синограммы. Для усиления художественного эффекта изображение лучше ограничить вписанной в границы изображения окружностью, а также применить операцию контрастирования с «зашкаливанием» небольшого числа пикселей с высокой яркостью. Пример такой визуализации приведен на Рис. 2.5.

Сдается программа, строящая схему расположения нитей с использованием алгоритма проецирования и свертки, а также визуализирующая программа. Необходимо предложить варианты улучшения алгоритма.
