# Greedy-решение (высокое качество)

Алгоритм вычислительной ниткографии с **жадным выбором линий по остатку**.

## Описание алгоритма

Экспериментальный алгоритм, превосходящий FBP по качеству:

1. **Nail Catalogue** — генерация хорд между гвоздями на окружности
2. **Sparse Matrix** — предвычисленная матрица пиксель × линия
3. **Greedy Selection** — на каждом шаге выбирается линия с максимальным вкладом в ошибку
4. **Residual Update** — обновление остатка `target - current`
5. **Early Stopping** — остановка при насыщении ошибки

## Преимущества над FBP

✅ **RMSE 0.191** (в 2.4× лучше FBP)
✅ **SSIM 0.997** (почти идеальное сходство)
✅ Физическая модель (гвозди на окружности)
✅ Без случайного шума (dithering)
✅ Адаптивное число линий

## Запуск

```bash
# Базовый запуск (300 гвоздей, 8000 линий)
python3 build_schema.py input.png --nails 300 --min-nail-gap 10 --lines 8000 --darkness 0.02 --out out

# Быстрый запуск (256 гвоздей, 6000 линий)
python3 build_schema.py input.png --nails 256 --min-nail-gap 14 --lines 6000 --darkness 0.03 --out out
```

## Файлы

- `build_schema.py` — основной скрипт
- `string_art.py` — nail catalogue, greedy selection
- `metrics.py` — комплексные метрики качества
- `visualize_schema.py` — визуализация схемы
- `run.sh` — запуск с параметрами по умолчанию
- `out_example/` — пример результата

## Параметры

- `--nails` — число гвоздей на окружности (default: 256)
- `--min-nail-gap` — минимальная дистанция между гвоздями (default: 14)
- `--lines` — максимум линий (default: 4000)
- `--darkness` — вклад одной нити (default: 0.03)
- `--min-score` — порог остановки (default: 0.0001)
- `--out` — папка результата (default: out)

## Выходные файлы

- `schema.csv` — схема нитей (nail_a, nail_b, angle_deg, rho_px)
- `preview.png` — растровое превью (×3, напр. 1311×1311 px)
- `preview.svg` — векторное превью (бесконечный зум)
- `compare.png` — сравнение оригинал vs результат
- `maps_error.png` — карта ошибки
- `maps_heatmap.png` — тепловая карта
- `metadata.json` — параметры запуска + зоновые метрики
- `metrics_report.txt` — детальный отчёт по метрикам

## Метрики качества

- **MSE/RMSE/MAE** — стандартные метрики ошибки
- **SSIM** — структурное сходство
- **PSNR** — отношение сигнал/шум (дБ)
- **Pearson Correlation** — линейная связь
- **Gradient Error** — качество границ
- **Зоновые метрики** — центр (лицо) vs кольцо (фон)
