# FBP-решение (по заданию)

Алгоритм вычислительной ниткографии на основе **Filtered Back-Projection** (FBP).

## Описание алгоритма

Соответствует заданию (task.md, раздел 2.2):

1. **Radon Transform** — преобразование Радона для заданного числа углов
2. **Ramp Filter** — свертка каждой проекции с рамп-фильтром
3. **Thinning** — прореживание до заданного числа нитей на угол
4. **Normalization** — зануление отрицательных значений, нормировка к [0,1]
5. **Random Dithering** — бинарное распыление sinogram

## Запуск

```bash
# Рекомендуемый запуск (по умолчанию в run.sh)
python3 build_schema.py input.png --angles 360 --threads 40 --seed 1 --brightness-lift 0.35 --out out

# Error diffusion вместо random dither
python3 build_schema.py input.png --angles 180 --threads 20 --seed 1 --dither-mode error-diffusion --out out
```

**Лучшая конфигурация**: 90 углов × 10–20 нитей на угол

## Файлы

- `build_schema.py` — основной скрипт
- `string_art.py` — функции Radon, фильтрации, дизеринга
- `visualize_schema.py` — визуализация схемы
- `run.sh` — запуск с параметрами по умолчанию
- `out_example/` — пример результата

## Параметры

- `--angles` — число углов (default: 180)
- `--threads` — максимум нитей на угол (default: 20)
- `--seed` — зерно случайного распыления (default: 1)
- `--dither-mode` — random или error-diffusion (default: random)
- `--out` — папка результата (default: out)

## Выходные файлы

- `schema.csv` — схема нитей (angle_deg, rho_px)
- `preview.png` — растровое превью (×3, тёмные детали = темнее, без «негатива»)
- `preview.svg` — вектор: белые нити на чёрном (реальная картина из ниток)
- `01_sinogram.png` — преобразование Радона
- `02_filtered.png` — после рамп-фильтра
- `03_probabilities.png` — после нормировки
- `04_binary_sinogram.png` — после дизеринга
- `metadata.json` — параметры запуска + RMSE/MAE

## Ограничения

- Случайный дизеринг вносит шум
- Непрерывные линии (angle, ρ) не соответствуют гвоздям на окружности
- Требует много углов для тонких деталей
- FBP принципиально не достигает RMSE < 0.4

Для **высокого качества** используйте `greedy_solution/`.
